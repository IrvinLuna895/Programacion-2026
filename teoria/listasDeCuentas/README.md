Sistema Bancario Rockerin — Actualización (Lista de Cuentas y Validación)

En esta versión se implementa una relación de uno a muchos, donde un cliente puede tener múltiples cuentas mediante el uso de una lista. Se agregan métodos en la clase Cliente para gestionar estas cuentas, como agregar, eliminar y recuperar por índice.

Además, se mejora la validación de entradas en el menú, asegurando que el usuario solo pueda seleccionar cuentas válidas y obligándolo a ingresar datos correctos mediante reintentos. También se muestran únicamente las cuentas disponibles al momento de elegir.

Estos cambios hacen que el sistema sea más robusto, escalable y alineado con buenas prácticas de diseño en programación orientada a objetos.
