Sistema de Biblioteca — Actualización Integral

Esta versión del proyecto de biblioteca incorpora todos los conceptos trabajados previamente en los ejercicios del sistema bancario, actualizando su diseño hasta el punto de listas de objetos y validación robusta.

Se implementa encapsulamiento mediante atributos privados y acceso controlado con getters. Los métodos __str__ sustituyen a funciones de visualización tradicionales para representar objetos como cadenas. Además, se aplica separación entre componentes Entidad y Contacto: las validaciones lógicas devuelven valores booleanos y la interacción con el usuario se concentra en el menú.

El sistema incluye relaciones entre objetos, donde una biblioteca administra una lista de libros (relación uno a muchos) y un cliente mantiene una lista de préstamos. También se agregan métodos especializados para manipular estas listas, como agregar, eliminar y recuperar elementos.

El proyecto queda actualizado hasta el tema de listas de objetos, integrando encapsulamiento, relaciones entre clases, validaciones, objetos como cadenas y colecciones dinámicas en un mismo sistema funcional y escalable.
